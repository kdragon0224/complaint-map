# BRD — 도로 관리주체 확인앱
> Business Requirements Document  
> 작성: 한국도로공사 전북본부 | 최초 작성: 2026년 6월 | 최종 수정: 2026년 7월

---

## 1. 배경 및 목적

### 문제 인식
한국도로공사 콜센터 및 지사로 접수되는 민원은 **관할 민원과 그외 민원이 혼재**되어 있다.  
민원 담당자가 접수 위치를 지도에서 확인하고 담당 기관을 찾는 데 시간이 소요되며,  
민자도로·국도·지방도·시군도의 경우 관리주체를 별도로 확인해 안내해야 하는 번거로움이 있다.

### 목적
민원 접수 위치를 입력하면 **담당 관리주체를 즉시 확인**할 수 있도록 한다.  
고속도로뿐 아니라 **어느 위치를 찍어도 반드시 안내가 나오는 것**("정보 없음" 제로)을 목표로 한다.

### 기대 효과
- 민원 배부 시간 단축
- 관할 오배부 감소
- 민자도로·타기관 민원 안내 오류 감소

---

## 2. 사용자 및 사용 시나리오

### 주 사용자
- 한국도로공사 전북본부 민원 담당 직원
- 각 지사 민원 접수 담당자

### 사용 시나리오
1. 민원 전화 접수 → 민원 발생 위치(지명, 주소, IC·JC명) 청취
2. 앱에 위치 입력 → 지도 핀 확인 및 위치 보정
   - PC: 핀 드래그 또는 우클릭
   - 스마트폰: 지도를 움직여 중앙 고정핀으로 위치 지정
3. 담당 기관명·도로유형·노선 즉시 확인
4. 해당 기관으로 민원 전달 (관할 외 민원은 해당 기관 안내)

## 3. 관리주체 판정 체계 (핵심)

**"도로 인식"과 "관리주체 판정"을 분리**한 5계층 구조.  
위에서부터 순서대로 매칭하며, 마지막 계층이 반드시 응답한다.

| 계층 | 도로 유형 | 관리주체 판정 근거 | 표시 예 |
|------|----------|------------------|---------|
| 1 | 고속국도 (도공) | **직제세부운영계획** (내부 문서, 연 1회 갱신) | 한국도로공사 전북본부 전주지사 |
| 2 | 고속국도 (민자) | 노선→운영사 테이블 (19개 등록) | 천안논산고속도로(주) |
| 3 | 일반국도 | **도로법 제23조 규칙**: 특별·광역시 및 시(市) 동지역 → 해당 시 / 그 외 → 지방국토관리청 | 전주시 (시 관리 국도 구간) / 익산지방국토관리청 |
| 4 | 지방도 | 시 동지역 → 시 / 그 외 → 도 (제주는 도 직접 관리) | 전북특별자치도 (도로관리부서) |
| 5 | 도시고속화도로·간선·기타 | 이름→기관 테이블 + 시군구 폴백 | 서울특별시·성남시 (구간별 관리) / 평창군 도로관리부서 |

> 도공 관할 구간만 내부 자료(직제)가 필수이고, 나머지는 법령·공개정보로 판정 가능하다는 원칙.

## 4. 기능 요구사항

| 번호 | 기능 | 우선순위 | 구현 상태 |
|------|------|----------|-----------|
| F-01 | 주소/지명/IC·JC명으로 위치 검색 | 필수 | ✅ 완료 |
| F-02 | 카카오맵 핀 표시 + 위치 보정 (PC: 드래그·우클릭 / 모바일: 중앙 고정핀) | 필수 | ✅ 완료 |
| F-03 | 담당 관리기관 표시 (전 도로 유형, 정보없음 제로) | 필수 | ✅ 완료 |
| F-04 | 도로유형 배지 + 노선명(고속도로는 이정 km 포함) 표시 | 필수 | ✅ 완료 |
| F-05 | 핀~관리노선 이격거리 경고 (200m 초과 시) | 권장 | ✅ 완료 |
| F-06 | 오류 신고 게시판 (관리자 삭제 기능 포함) | 권장 | ✅ 완료 |
| F-07 | 이용 현황 통계 (조회 건수, 시간대별, 기관별 TOP 10) | 권장 | ✅ 완료 |
| F-08 | 카카오톡 링크 공유 시 앱 정보 미리보기 (OG 태그) | 권장 | ✅ 완료 |
| F-09 | 고속도로 시설(IC·JC·TG 등) 검색 시 고속국도 우선 판정 | 권장 | ✅ 완료 |

> 전화번호·사무실 주소 표시 기능은 응답 지연(전체의 94~97%)으로 **2026.6 제거**.

## 5. 결과 카드 표시 항목

```
[기관명]              예) 한국도로공사 전북본부 전주지사
[도로유형배지] [노선명]  예) 고속국도  논산천안선/호남선 169.5km 지점
                        예) 일반국도  국도 21호선 새만금북로
────────────────────────────────
⚠️ 핀이 관리노선에서 XXXm 떨어져 있습니다  ← 200m 초과 시만 표시
```

- 도로유형 배지 색: 고속국도(녹색) / 일반국도(파랑) / 지방도(보라) / 도시고속화도로(주황) / 시군도(회색)
- 도공 외 기관은 카드 배경이 황색 계열로 구분

## 6. 데이터 기준

### 고속도로 (계층 1·2)
- **관할 구간**: 한국도로공사 2026년 직제세부운영계획 (hwpx, 내부 문서) — 9개 지역본부 체계
- **도로 좌표**: ETC 도로중심선 CSV + 민자 보간 포인트 (누락 13개 노선 보완 완료)
- **민자 운영사**: 민자도로관리지원센터(CEPHIS) 및 공식 자료 기준, 19개 코드 등록

### 그 외 도로 (계층 3~5)
- **도로 좌표·등급·번호·이름**: OpenStreetMap(OSM) 한국 전체 데이터에서 추출  
  (국도·지방도·주요 간선 127,113개 포인트, 0.5km 간격)
- **행정구역**: 카카오 행정구역 API (시도·시군구·동)
- **판정 규칙**: 도로법 제23조 (코드로 구현, `lib/road-rules.ts`)

### 데이터 소스 선택 근거
| 후보 | 결론 |
|------|------|
| OSM | ✅ 채택 — 즉시 다운로드, 갱신 자동화, 신설도로 반영 빠름 |
| V-World (국토지리정보원) | 보류 — 공신력은 높으나 로그인·수동 다운로드 필요. 오분류 신고 누적 시 검증용으로 활용 |

## 7. 품질 검증

자동 샘플링 테스트로 검증 (실무자 베타 전 사전 검증):

| 회차 | 규모 | 결과 |
|------|------|------|
| 1차 (고속도로+국도 시절) | 410건 | 누락 민자 13개 노선 발견 → 보완 후 100% |
| 2차 (신규 프로세스) | 555건 | 신행정구역·제주 특례·민자 지방도 발견 → 보완 |
| 3차 (최종) | 318건 | **100% 통과** |

응답 속도: 핀 이동 31~48ms, 주소 검색 40~48ms.

## 8. 비기능 요구사항

- 모바일(스마트폰) 우선 UI — 중앙 고정핀 방식
- 인터넷 연결 환경에서만 사용 (오프라인 불필요)
- 별도 로그인 없이 즉시 사용 가능
- 검색 응답속도 2초 이내 목표 (실측 0.05초 수준으로 달성)

## 9. 운영 및 유지보수

| 상황 | 조치 |
|------|------|
| 조직 개편 (지사 신설·폐지) | `lib/highway-jurisdiction.ts` → `JURISDICTION_RULES` 수정 (연 1회) |
| 민자 고속도로 운영사 변경·신설 | `lib/highway-jurisdiction.ts` → `PRIVATE_ROAD_CODES` 수정 |
| 도시고속화도로 오류 신고 | `lib/road-rules.ts` → `URBAN_EXPRESSWAYS` 테이블에 한 줄 추가 |
| 민자 지방도·교량 추가 | `lib/road-rules.ts` → `PRIVATE_LOCAL_ROADS` 테이블 수정 |
| 국도·지방도 데이터 갱신 | OSM 최신 데이터 다운로드 → 변환 스크립트 재실행 (자동화 가능) |
| 오류 신고 게시판 관리 | `/feedback` 페이지, 관리자 비밀번호로 삭제 |
| 이용 현황 확인 | `/stats` 페이지, 관리자 비밀번호 |

> 모든 코드 변경 후 `git push origin main` 시 Vercel이 자동 배포한다.
