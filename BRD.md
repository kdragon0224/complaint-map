# BRD — 도로 관리주체 확인앱
> Business Requirements Document  
> 작성: 한국도로공사 전북본부 | 최초 작성: 2026년 6월

---

## 1. 배경 및 목적

### 문제 인식
한국도로공사 콜센터 및 지사로 접수되는 민원 중 상당수가 **관할 외 민원**이다.  
민원 담당자가 접수 위치를 지도에서 확인하고 담당 지사를 찾는 데 시간이 소요되며,  
민자도로 구간의 경우 운영사를 별도로 안내해야 하는 번거로움이 있다.

### 목적
민원 접수 위치를 입력하면 **담당 관리주체(지사 또는 민자 운영사)와 연락처를 즉시 확인**할 수 있도록 한다.

### 기대 효과
- 민원 배부 시간 단축
- 관할 오배부 감소
- 민자도로 안내 오류 감소

---

## 2. 사용자 및 사용 시나리오

### 주 사용자
- 한국도로공사 전북본부 민원 담당 직원
- 각 지사 민원 접수 담당자

### 사용 시나리오
1. 민원 전화 접수 → 민원 발생 위치(지명, 주소, IC·JC명) 청취
2. 앱에 위치 입력 → 지도 핀 확인 및 위치 보정
3. 담당 기관명·전화번호·이정(km) 즉시 확인
4. 해당 기관으로 민원 전달

## 4. 데이터 기준

### 관할 구간 기준
- **출처**: 한국도로공사 2026년 직제세부운영계획 (hwpx)
- **기준 조직**: 9개 지역본부 체계
  - 수도권본부 (2025.3 신설): 인천·시흥·군포·화성·파주지사
  - 서울경기본부: 수원·경기광주·동서울·이천·용인지사
  - 전북본부, 광주전남본부, 대전충남본부, 대구경북본부, 부산경남본부, 강원본부

### 민자도로 기준
- 한국도로공사 직제세부운영계획에 없는 구간 = 민자 구간
- 민자 운영사는 민자도로관리지원센터(CEPHIS) 및 위키백과 기준

## 5. 비기능 요구사항

- 모바일(스마트폰) 우선 UI
- 인터넷 연결 환경에서만 사용 (오프라인 불필요)
- 별도 로그인 없이 즉시 사용 가능
- 검색 응답속도 2초 이내 목표
